Система автоматизации внеучебной деятельности колледжа. Проект позволяет студентам в один клик записываться на секции, кружки и управлять получением литературы в библиотеке через QR-коды.

Важное примечание: На текущем этапе реализована клиентская (студенческая) часть. Архитектура проекта спроектирована с учетом прямой синхронизации с Панелью Преподавателя (Admin Dashboard) для управления данными в реальном времени.

🛠 Основной функционал (Current Version)
1. Landing Page (Главный экран)
Минималистичный интерфейс для быстрого доступа к трем ключевым сервисам:

Взять книгу: Мгновенный запуск QR-сканера.

Секции: Переход в каталог спортивных направлений.

Кружки: Переход в каталог творческих и научных объединений.

Вход для персонала: Выделенная точка доступа к системе управления.

-----------------------------

1. Главная страница (Landing)
Минимализм и фокус на три основные кнопки. Никаких лишних меню, только чистый функционал.

Шапка (Header):

Слева: Название ATC leisure.

Справа: Кнопка «Для персонала» (ведет на login/teacher).

Центральный блок (Hero):

Кнопка «Взять книгу» (самая заметная, открывает камеру для QR).

Кнопка «Присоединиться к секции» (переход на /sections).

Кнопка «Кружки» (переход на /clubs).

2. Разделы: Секции и Кружки
Страницы /sections и /clubs работают по идентичному шаблону:

Верхняя панель управления:

Поисковик (Input): «Поиск по названию...» — для быстрой фильтрации карточек.

Кнопка «Зарегистрироваться»: Большая кнопка справа от поиска, ведущая на общую форму регистрации.

Сетка карточек (Grid):

Каждая карточка — это название (например, «Футбол» или «Шахматы») и яркая картинка.

Взаимодействие с картой (Click/Expand):
При клике на карточку она раскрывается (или вылетает модальное окно), показывая:

Расписание: (например: Пн, Ср, Пт — 17:00).

Руководитель: ФИО + Номер телефона (кликабельный tel:).

Статистика: Общее количество участников (например: «Уже записано: 24 человека»).

3. Формы регистрации (Студенческий путь)
А. Форма «Запись на секцию/кружок»
Единая форма для сбора данных. Чтобы данные в базе были чистыми, используем выпадающие списки.

ФИО (текстовое поле).

Группа (текстовое поле или список).

Куратор (текстовое поле).

Номер телефона (маска ввода +7 (XXX) ...).

Выбор направления: Выпадающий список (Dropdown), где подтягиваются все доступные секции (или кружки) из базы.

Б. Библиотечная форма (после QR)
Появляется только после того, как камера успешно считала код.

ФИО.

Группа.

Название книги (может заполняться автоматически из QR-кода).

Количество.

Кнопка «Подтвердить».

-- 1. Создаем таблицу для кружков и секций
CREATE TABLE activities (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  type TEXT NOT NULL CHECK (type IN ('section', 'club')), -- 'section' для секций, 'club' для кружков
  title TEXT NOT NULL,
  description TEXT,
  schedule TEXT,          -- Расписание (например: "Пн, Ср 17:00")
  mentor_name TEXT,       -- ФИО руководителя
  mentor_phone TEXT,      -- Номер телефона руководителя
  image_url TEXT,         -- Ссылка на картинку
  total_participants INT DEFAULT 0 -- Общее количество участников (для счетчика)
);

-- 2. Создаем таблицу регистраций студентов на кружки/секции
CREATE TABLE registrations (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  student_name TEXT NOT NULL,
  group_name TEXT NOT NULL,
  curator_name TEXT,
  phone TEXT,
  activity_id UUID REFERENCES activities(id) ON DELETE CASCADE -- Связь с таблицей активностей
);

-- 3. Создаем таблицу книг
CREATE TABLE books (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  title TEXT NOT NULL,
  author TEXT,
  isbn TEXT
);

-- 4. Создаем таблицу заявок в библиотеку
CREATE TABLE library_requests (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  student_name TEXT NOT NULL,
  group_name TEXT NOT NULL,
  book_id UUID REFERENCES books(id) ON DELETE CASCADE, -- Связь с конкретной книгой
  quantity INT DEFAULT 1,
  status TEXT DEFAULT 'pending' CHECK (status IN ('pending', 'issued', 'returned')) -- Статусы для админки
);
